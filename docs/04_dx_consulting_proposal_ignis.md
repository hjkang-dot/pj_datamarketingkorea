# 1차 DX 컨설팅 제안서: 이그니스

## 1. 제안 개요

본 제안서는 1차 아웃바운드 타깃인 이그니스의 D2C 브랜드 성장에 맞춰 CRM, 마케팅 데이터, 캠페인 리포팅 운영을 고도화하기 위한 DX 컨설팅 제안입니다.

이그니스는 랩노쉬, 닥터랩노쉬, 한끼통살 등 복수 브랜드를 운영하는 푸드/건강기능식품 기반 D2C 커머스 기업으로 판단됩니다. 브랜드와 제품군이 확장될수록 고객 세그먼트, 광고 성과, 재구매 데이터, 브랜드별 KPI를 통합적으로 관리하는 체계가 중요해집니다.

본 컨설팅은 공개 정보 기반의 사전 가설을 출발점으로 하며, 실제 프로젝트 착수 시 내부 데이터와 현업 인터뷰를 통해 진단 결과를 보정합니다.

## 2. 제안 배경

### 2.1 시장 및 운영 환경

건강기능식품과 D2C 커머스 시장은 광고 효율, 고객 재구매, 제품별 구매 주기, 채널별 성과 관리가 매출 성장을 좌우합니다. 특히 멀티 브랜드를 운영하는 기업은 브랜드별 성장 지표를 따로 보는 것뿐 아니라, 공통 고객 기반과 캠페인 성과를 함께 해석해야 합니다.

### 2.2 이그니스에 대한 문제 가설

| 영역 | 문제 가설 | 근거 신호 |
| --- | --- | --- |
| CRM | 고객군별 구매 주기와 재구매 패턴이 브랜드별로 다를 가능성 | RFM 세그먼트, 타깃 캠페인 관련 CRM 채용 신호 |
| 데이터 통합 | 자사몰, 광고, 상품, 브랜드 데이터가 분산되어 있을 가능성 | D2C 멀티 브랜드 운영 구조 |
| 리포팅 | 캠페인 성과 보고와 브랜드별 KPI 취합에 수작업이 존재할 가능성 | 마케팅 운영 규모 확대 가능성 |
| 리텐션 | 신규 고객 확보 이후 재구매와 이탈 방지 체계가 중요할 가능성 | 건강기능식품/푸드 제품군 특성 |
| 의사결정 | 브랜드별 성과를 같은 기준으로 비교하기 어려울 가능성 | 랩노쉬, 닥터랩노쉬, 한끼통살 등 브랜드 포트폴리오 |

## 3. 컨설팅 목표

본 프로젝트의 목표는 “마케팅 성과를 더 빠르게 보고, 고객 세그먼트별로 더 정교하게 실행하며, 반복 리포팅 업무를 줄이는 것”입니다.

| 목표 | 설명 |
| --- | --- |
| CRM 운영 진단 | 고객 세그먼트, 구매 주기, 재구매 흐름, 캠페인 운영 현황을 진단 |
| KPI 체계 정리 | 브랜드별/채널별 성과 지표를 공통 기준으로 정리 |
| 리포팅 자동화 | 주간/월간 캠페인 성과 취합 업무를 자동화할 수 있는 구조 설계 |
| 대시보드 설계 | ROAS, CAC, LTV, 재구매율, 브랜드별 매출 기여도를 한 화면에서 관리 |
| 90일 실행 로드맵 | 빠른 성과 과제와 중장기 데이터 고도화 과제를 분리 |

## 4. 컨설팅 범위

| 구분 | 주요 활동 | 산출물 |
| --- | --- | --- |
| 현황 진단 | 현업 인터뷰, 데이터 소스 파악, 캠페인 운영 흐름 분석 | DX 현황 진단 리포트 |
| 데이터 진단 | 고객, 주문, 광고, 상품, 브랜드 KPI 구조 확인 | 데이터 맵, KPI 정의서 |
| CRM 분석 | RFM, 구매 주기, 이탈 위험군, 고가치 고객군 분석 | 고객 세그먼트 분석 리포트 |
| 리포팅 개선 | 반복 보고서, 수작업 취합, 지표 불일치 구간 파악 | 리포팅 자동화 개선안 |
| 대시보드 설계 | 경영진/마케팅팀 관점의 핵심 지표 설계 | 미니 대시보드 프로토타입 |
| 실행 로드맵 | 우선순위, 일정, 담당 역할, 기대 효과 정리 | 30/60/90일 실행 계획 |

## 5. 제안 과제

### 과제 1. CRM 고객 세그먼트 진단

고객을 최근 구매일, 구매 빈도, 구매 금액, 제품군, 브랜드 접점 기준으로 분류합니다. 이를 통해 신규 고객, 반복 구매 고객, 휴면 위험 고객, 고가치 고객을 구분하고 각 세그먼트별 캠페인 방향을 제안합니다.

| 분석 항목 | 활용 방안 |
| --- | --- |
| RFM 세그먼트 | 고객군별 메시지와 혜택 차등화 |
| 구매 주기 | 재구매 리마인드 캠페인 타이밍 설계 |
| 브랜드 교차 구매 | 랩노쉬, 닥터랩노쉬, 한끼통살 간 업셀/크로스셀 기회 발굴 |
| 휴면 위험군 | 이탈 방지 캠페인 대상 정의 |

### 과제 2. 마케팅 KPI 통합 대시보드 설계

자사몰, 광고, 주문, 브랜드 데이터를 연결해 핵심 지표를 한 화면에서 볼 수 있는 대시보드를 설계합니다.

| KPI | 설명 |
| --- | --- |
| ROAS | 광고비 대비 매출 성과 |
| CAC | 신규 고객 획득 비용 |
| LTV | 고객 생애 가치 |
| 재구매율 | 일정 기간 내 반복 구매 고객 비율 |
| 브랜드별 매출 기여도 | 브랜드 포트폴리오 성과 비교 |
| 캠페인별 전환율 | CRM/광고 캠페인의 전환 효율 |

### 과제 3. 캠페인 리포팅 자동화

주간/월간 캠페인 리포트 작성 과정에서 반복되는 데이터 다운로드, 취합, 지표 계산, 시각화 작업을 자동화 대상으로 정의합니다.

| 개선 전 | 개선 후 |
| --- | --- |
| 매체별 데이터를 수동 다운로드 | 데이터 소스별 자동 취합 구조 설계 |
| 엑셀 기준 지표 계산 | KPI 정의서 기반 자동 계산 |
| 브랜드별 보고 형식 상이 | 공통 템플릿 기반 리포팅 |
| 회의 전 수작업 검증 | 대시보드 기반 상시 확인 |

### 과제 4. 브랜드별 리텐션 전략 수립

브랜드별 제품 특성과 구매 주기에 맞춰 리텐션 전략을 분리합니다. 건강기능식품성 제품은 섭취 주기와 재구매 타이밍을, 푸드 제품은 반복 구매와 식단 목적성을 중심으로 분석합니다.

| 브랜드 유형 | 리텐션 전략 예시 |
| --- | --- |
| 건강기능식품 | 섭취 주기 기반 재구매 리마인드 |
| 식사 대체/간편식 | 반복 구매 패턴 기반 번들 제안 |
| 단백질/고단백 식품 | 목표 고객군별 챌린지 캠페인 |
| 신규 브랜드 | 첫 구매 후 14일/30일 온보딩 캠페인 |

## 8. 필요 데이터

실제 진단을 위해 다음 데이터가 필요합니다.

| 데이터 | 주요 필드 예시 |
| --- | --- |
| 주문 데이터 | 주문일, 고객 ID, 상품명, 브랜드, 주문 금액, 수량 |
| 고객 데이터 | 가입일, 유입 채널, 성별/연령대, 멤버십 여부 |

| CRM 캠페인 데이터 | 발송일, 세그먼트, 메시지, 오픈율, 클릭률, 구매 전환 |
| 상품 데이터 | 브랜드, 카테고리, 가격, 마진, 출시일 |

## 9. 프로젝트 성공 조건

- 브랜드별 핵심 KPI 정의에 대한 내부 합의가 필요합니다.
- 고객 데이터와 주문 데이터를 연결할 수 있는 기준 키가 필요합니다.
- CRM 캠페인 성과를 구매 데이터와 연결할 수 있어야 합니다.
- 초기에는 모든 데이터를 완벽히 통합하기보다, 의사결정에 중요한 핵심 지표부터 시작해야 합니다.

## 10. 제안 결론

이그니스는 멀티 브랜드 D2C 성장 구조를 가지고 있어 CRM, 마케팅 데이터, 리포팅 운영을 고도화할 여지가 큽니다. 본 컨설팅은 거대한 시스템 전환보다 고객 세그먼트 진단, KPI 정리, 대시보드 프로토타입, 리포팅 자동화처럼 빠르게 성과를 확인할 수 있는 과제부터 추진하는 것을 제안합니다.

초기 4주 진단을 통해 개선 과제를 구체화하고, 이후 90일 동안 CRM 실행 체계와 마케팅 KPI 대시보드를 구축하면 데이터 기반 마케팅 운영 수준을 한 단계 높일 수 있습니다.

## 11. 참고 및 한계

- 본 제안서는 공개 정보 기반의 사전 제안서입니다.
- 실제 컨설팅 착수 전에는 최신 채용공고, 브랜드별 판매 채널, 내부 데이터 구조, 마케팅 운영 도구를 재확인해야 합니다.
- 정량 기대 효과는 내부 데이터 확인 후 산정해야 합니다.
- 분석 리포트와 미니 대시보드는 공개 데이터가 더 풍부한 프롬바이오를 최종 분석 타깃으로 전환해 제작합니다.
